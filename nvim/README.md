# Neovim

[LazyVim](https://www.lazyvim.org/) base + a thin custom layer for
competitive programming. Symlinked into `~/.config/nvim` from this repo,
so editing files here edits the live config.

- **Directory map and what's enabled** — [`docs/README.md`](docs/README.md)
- **Task-oriented C/C++ guide** (run, test, debug, format) — [`docs/cpp.md`](docs/cpp.md)
- **Keymaps this config adds on top of LazyVim** — [`docs/keymaps.md`](docs/keymaps.md)
- **Competitive-programming workflow** (CompetiTest + custom runner) — [`docs/competitive-programming.md`](docs/competitive-programming.md)
- **Debugging via DAP** — [`docs/debugging.md`](docs/debugging.md)
- **LSP setup** — [`docs/lsp.md`](docs/lsp.md)
- **Gotchas worth knowing** — [`docs/troubleshooting.md`](docs/troubleshooting.md)

LazyVim's own keymap reference is the source of truth for everything not
in [`docs/keymaps.md`](docs/keymaps.md): <https://www.lazyvim.org/keymaps>.
